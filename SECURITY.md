# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in UNIFY, please report it responsibly. **Do not** create a public GitHub issue for security vulnerabilities.

### How to Report

Please email security concerns to: **twumgyanchristian2@gmail.com**

Include the following information:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- You will receive an acknowledgment within 48 hours
- We will investigate and provide updates regularly
- Once fixed, we will coordinate disclosure timing with you

## Security Best Practices

When using UNIFY:

1. **Keep Dependencies Updated** - Regularly update your dependencies
2. **Use HTTPS** - Always use HTTPS when accessing the application
3. **Protect Credentials** - Never commit API keys or sensitive data
4. **Report Issues Responsibly** - Follow the reporting guidelines above
5. **Stay Informed** - Follow security advisories and updates
